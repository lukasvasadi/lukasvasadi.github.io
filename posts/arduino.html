Introduction

Arduino is an opensource platform for microcontroller boards. A microcontroller, or microcontroller unit (MCU), is an integrated circuit that contains the processor, memory, and input/output (I/O) bundled together in one package. This is different from the more commonly known central processing unit (CPU), which contains the processor without any peripherals. Microcontrollers are used to control sensors and actuators that interact with the environment.

Code Structure

"Blink" is one of the simplest source code examples (see below) that demonstrates the structure of an MCU program. The Arduino interactive development environment (IDE), i.e., the software user interface where you type code, uses an application programming interface (API) to simplify some of the C/C++ programming. In simple terms, you—the developer—always follow the same procedure below: (1) declare global variables, (2) run setup once, (3) loop through main function indefinitely.

In this case, we are first declaring a global variable led that represents the I/O pin header number on the Arduino board. The variable is preceded by a datatype indicator, int, that reserves a specific amount of onchip memory. Next, in the setup function, the led pin is configured as a digital OUTPUT, meaning that the MCU will perform a write operation to that pin, as opposed to a read operation. In the C/C++ programming languages, as well as the Arduino API, a function must be declared with a datatype that references its return value. For example, a function that performs a set of calculations and passes an integer value back to the main loop must be declared int. If the function has no return value, then it is declared void.

During compilation, where the code is translated into machine language, the Arduino IDE knows to execute the setup function once and the loop repeatedly. Within the loop function, the digital pin, which is connected to a physical LED on the board, is toggled HIGH and LOW with the built-in digitalWrite function. For humans to see the difference, we must add a delay between function calls, in this case 1000 micro-seconds.

When this code is compiled and "flashed," or loaded, onto the MCU, you should see the onboard green LED flash on/off at one-second intervals.

/*

Blink

Switches LED on for one second, then off for one second, repeatedly

*/

int led = 13;

void setup() {

// set led pin as output

pinMode(led, OUTPUT);

}

void loop() {

// toggle led on and off

digitalWrite(led, HIGH);

delay(1000);

digitalWrite(led, LOW);

delay(1000);

}
