Building an Electron app with React

Last updated: June 2022

Contents

  • Introduction
  • Setup

Introduction

This guide assumes some familiarity with modern JavaScript and Node.js. To install Node.js and its package manager (npm), follow the instructions from the environment setup tutorial. Some installations will require the use of yarn:

npm i --global yarn

confirm installation

yarn --version

The following tutorial is based on a YouTube video by Coding with Justin. The information has been condensed and, in some cases, clarified for a lay audience.

Electron is a framework for creating desktop applications with web development tools, i.e., JavaScript, HTML/CSS. It is developed and maintained by GitHub, and supports some of the most popular modern desktop applications, including Slack and VSCode. The frontend is rendered using the Chromium browser engine—the same engine powering Google Chrome—while the backend uses the Node.js JavaScript runtime environment.

React is primarily a frontend JavaScript library, which was created by Facebook (Meta) in 2013. With React, developers build UI elements as "components," which maintain a state that can be controlled independent of the Document Object Manager (DOM), i.e., UI components can be updated individually, without refreshing the entire page. In my opinion, this is the most compelling reason to use React alonside Electron, especially when creating applications with high rendering demands, such as a video player.

Setup

Electron and React each require a dedicated process.

To start, configure a React application with the create-react-app command line tool:

npx create-react-app project-dirname

Navigate to the new project directory and install the dependencies needed to bridge Electron and React:

cd project-dirname

yarn add -D concurrently cross-env electron electron-builder electronmon wait-on
