Programming STM32 dev boards with VSCode

Last updated: December 2021

Contents

  • Introduction
  • Environment setup

Introduction

This guide will explain how to set up an environment for programming an STM32 microcontroller (MCU). The website provides a listing of various MCUs for special use cases, such as "High Performance" or "Ultra Low Power." Search the "Mainstream MCUs" if unsure about the requirements.

Environment setup

Visual Studio Code

Visual Studio Code (VSCode) will be used as the main text editor. Within the VSCode extensions tab, install the Microsoft C/C++ and Cortex-Debug plugins.

GNU Arm embedded toolchain

On Windows machines, the GNU embedded toolchain for Arm microcontrollers can be installed via the executable from armDeveloper. Linux users can install the toolchain directly from the command line:

sudo apt install gcc-arm-none-eabi

For debugging, Linux users can install the GNU debugger:

sudo apt install gdb-multiarch

ST-LINK utility

ST-LINK is a software interface for programming STM32 microcontrollers. It allows developers to read and write programs and flash the microcontroller with a full-featured GUI or CLI. To install ST-LINK on Windows, download the executable from st.com (an email address is required to download the software). On Linux:

sudo apt install st-utils

OpenOCD

The Open On-Chip Debugger (OpenOCD) is a piece of software that helps debug programs after flashing the microcontroller.

sudo apt install openocd
